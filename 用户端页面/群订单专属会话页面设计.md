## 核心设计点
1. **会话专属属性**：仅用于群订单沟通，与个人商家会话相互独立
2. **一键同步功能**：商家消息可快速同步至对应群聊，无需手动选择
3. **订单信息集成**：实时展示群订单号、总价、成员、已点菜品、支付方式和状态
4. **同步状态反馈**：系统自动反馈消息同步结果
5. **权限控制**：仅群发起者和商家可查看该会话
6. **提示信息**：明确告知用户该会话的用途，避免与个人订单混淆

## 交互逻辑
1. 群订单创建并被商家接单后，系统自动创建该会话
2. 商家发送消息后，系统在消息右侧显示「同步到群」按钮
3. 发起者点击按钮后，系统自动提炼消息内容并同步至群聊
4. 同步完成后，系统在会话内反馈同步结果和状态
5. 会话内实时更新群订单的所有变化和状态
6. 群订单完成后，该会话自动归档至「群订单历史」

## 与群聊会话的关联机制
| 功能                | 群聊会话                          | 群订单专属会话                          |
|---------------------|-----------------------------------|---------------------------------------|
| 沟通对象            | 所有群成员                        | 仅群发起者与商家                        |
| 消息内容            | 群内所有聊天内容                  | 仅与群订单相关的内容                    |
| 同步功能            | 接收系统同步的商家消息            | 发送并同步商家消息至群聊                |
| 订单信息            | 展示简化的订单信息                | 展示完整的订单信息和状态                |
| 操作权限            | 所有群成员均可操作                | 仅发起者和商家可操作                    |
